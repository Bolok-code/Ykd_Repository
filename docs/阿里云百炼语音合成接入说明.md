# 阿里云百炼语音合成接入说明

## 当前实现范围

当前只实现机器人发送语音，不处理用户发来的语音：

```text
微信文字消息
→ ILinkMessageRouter 识别“用语音回答”意图
→ 天气服务或 DeepSeek 生成文字答案
→ BailianSpeechSynthesisClient 调用百炼非实时 TTS
→ 下载 24 kHz、单声道、16 位 PCM
→ NodeSilkSpeechEncoder 调用项目内 silk-wasm
→ 生成带 #!SILK_V3 文件头的 SILK
→ ILinkClient.sendVoice 以 encode_type=6 发送语音气泡
```

不能把 MP3 或 PCM 直接改名为 `.silk`。项目会验证 SILK 文件头、播放时长
和采样率，编码失败或 `sendVoice` 抛出异常时自动发送 WAV 文件；WAV 也发送
失败时才降级为文字。

注意：iLink 的 `sendVoice` 没有微信设备侧投递回执。日志显示“已提交给
iLink”只表示 SDK 调用完成，仍需使用手机微信验证是否出现语音气泡。
默认启用双发策略：提交 SILK 气泡后固定追加一份 WAV 文件，避免 iLink
静默丢弃原生语音时用户完全收不到音频。

## 安装 SILK 编码依赖

需要 Node.js 16.11 或更高版本。首次拉取项目后在项目根目录执行：

```powershell
npm --prefix tools/silk-encoder install --cache tools/silk-encoder/.npm-cache
```

验证编码器：

```powershell
npm --prefix tools/silk-encoder test
```

`node_modules` 和 `.npm-cache` 不提交 Git；`package.json` 与
`package-lock.json` 需要提交，确保其他组员安装相同版本。

## 配置

默认配置位于 `src/main/resources/application.yml`：

```yaml
app:
  ilink:
    voice:
      send-file-copy-after-native: ${ILINK_VOICE_SEND_FILE_COPY:true}
  bailian:
    tts:
      model: ${BAILIAN_TTS_MODEL:qwen-audio-3.0-tts-flash}
      voice: ${BAILIAN_TTS_VOICE:longanhuan_v3.6}
      sample-rate: ${BAILIAN_TTS_SAMPLE_RATE:24000}
      timeout-seconds: ${BAILIAN_TTS_TIMEOUT_SECONDS:180}
      silk:
        enabled: ${SILK_ENCODER_ENABLED:true}
        node-command: ${SILK_NODE_COMMAND:node}
        script-path: ${SILK_ENCODER_SCRIPT:tools/silk-encoder/encode.mjs}
        work-directory: ${SILK_ENCODER_WORK_DIR:work/silk}
        timeout-seconds: ${SILK_ENCODER_TIMEOUT_SECONDS:30}
```

语音合成复用现有的 `DASHSCOPE_API_KEY`、`BAILIAN_WORKSPACE_ID` 和
`BAILIAN_REGION`。Qwen-Audio-TTS 非实时接口需要北京地域。

## 测试方式

启动微信机器人后发送：

```text
请用语音回答：你好
```

也可以测试天气能力：

```text
请用语音回答：明天杭州天气
```

失败降级顺序：

```text
SILK 语音气泡 → WAV 文件 → 普通文字
```

## 错误诊断与重试

语音合成会解析百炼返回的 HTTP 状态、错误码、错误消息和 RequestId。
对网络异常、HTTP 429（非配额不足）和 HTTP 5xx 最多尝试三次，并采用
指数退避；API Key、权限、参数、余额或分配配额错误不会盲目重试。

最终失败日志示例：

```text
百炼语音合成提交合成任务失败 - httpStatus=429,
code=Throttling.AllocationQuota, requestId=xxx, message=xxx
```

可使用 RequestId 在百炼控制台或工单中定位对应请求。

百炼返回的 OSS 临时下载地址必须原样保留签名查询参数。项目仅把 `http`
协议替换为 `https`，不会再通过 URI 组件重新拼装 URL，避免 `%2B`、`%2F`
等字符被重新编码后触发 `SignatureDoesNotMatch`。OSS XML 错误中的 `Code`、
`Message` 和 `RequestId` 也会单独解析。
